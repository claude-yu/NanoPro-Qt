# 阶段 3 Qt 基准（实测，持续补充）

测试机：用户本机（Win11，Python 3.13.2，PySide6 6.11.1）。

## 首个数据点（2026-05-31，用户实跑）
| 场景 | 输入 | 结果 |
|---|---|---|
| 导入大图 | 一张大尺寸科研图(PNG) | **18 ms** |
| 多图层连续动画(压力测试) | **70 个图层**(大图叠加)持续平移动画 + 缩放至 40% | **FPS ≈ 173–188** |

> 截图证据：HUD 显示 `FPS 188.4 zoom 40%`、状态栏 `力测试: 70 项 · 导入 18 ms · FPS 173.6`。

## 对照门槛（PLAN.md 阶段3）
- 目标：4K + 20 图层拖动 ≈ 50 FPS。**实测 70 图层动画 ≈ 180 FPS → 远超目标（约 3.5×）。**
- 结论倾向：**交互/渲染性能 Qt 决定性胜出**（这是 WebView 全画布 JS 重绘最难优化的部分）。

## OpenCV 抠图算法耗时（2026-05-31，headless 实测 `benchmarks/benchmark_ops.py`）
OpenCV 4.13.0 · numpy 2.3.5。合成白底+前景块，点背景洪水填充。

| 尺寸 | 魔棒 floodFill | 羽化(GaussianBlur) | 提取 alpha | PNG 编码 |
|---|---|---|---|---|
| 1K 1024² | 8.4 ms | 0.6 ms | 3.6 ms | 5.2 ms |
| 2K 2048² | 20.4 ms | 1.0 ms | 16.2 ms | 17.9 ms |
| 4K 4096² | 80.8 ms | 1.3 ms | 62.5 ms | 70.3 ms |

**对照门槛全面碾压**：2K 魔棒 20ms(目标<300)、4K 魔棒 81ms(目标<1000)、4K 导出 ~130ms(目标<3000)。
→ 算法层 OpenCV(C/SIMD) 决定性快于现 JS/Worker（如预判）。

## 综合结论倾向（待补完整 UI 流程后定稿）
- **渲染/交互**：Qt 决定性胜（70 图层 ~180 FPS）。
- **算法**：OpenCV 决定性胜（4K 全流程 ~0.2s）。
- **能力**：图层上画笔/橡皮（WebView 做不到）Qt 天然支持。
- → 三项都强支持"渐进迁移 PySide6"。仍需：把 OpenCV 接入 UI 跑通交互式抠图+导出全流程，再写 `decision.md`。
